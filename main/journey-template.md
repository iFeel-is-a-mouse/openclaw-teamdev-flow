# journey.md — 项目过程日志

> **审计和复盘的关键依据。** 5 个 agent 均可写入。修改/删除/削弱任何条目需用户最高层同意。
>
> 使用方法：复制此模板到 `projects/<项目名>/docs/journey.md`，在项目推进过程中逐条追加。

---

## 第 1 条：项目启动

```markdown
## [时间] 阶段0 完成：复杂度评估与流程声明

📋 复杂度评估: [S/M/L]
📋 执行流程: [列出将执行的阶段]
📋 跳过/合并: [列出跳过的阶段及原因]

用户确认：✅
```

---

## 记录格式

每完成一个阶段，在下方追加一条记录：

```markdown
## [YYYY-MM-DD HH:mm] 阶段[N] 完成：[阶段名称]

**承接 Agent：** [agent 名称]
**状态：** ✅ 完成 / ⚠️ 有问题 / 🔁 退回

### 关键产出
- [产出物1]
- [产出物2]

### 关键决策
- [决策1及依据]
- [决策2及依据]

### 发现的问题
- [问题描述] → [处理方式] → [结果]
```
