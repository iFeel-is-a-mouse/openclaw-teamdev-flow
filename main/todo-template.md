# todo-template.md — 项目任务看板模板

> 复制此模板到 `projects/<项目名>/docs/todo.md`。
> **阶段结束标志：该阶段所有 todo 项全部 ✅。**
> 修改/删除/削弱任何条目需用户最高层同意。5 个 agent 均可编辑。

---

## 项目信息

| 字段 | 值 |
|------|-----|
| 项目名称 | `<项目名>` |
| 复杂度 | S / M / L |
| 流程阶段数 | `<N>` |
| 启动时间 | `YYYY-MM-DD HH:mm` |

## WBS 逐步求精

todo.md 从粗到细逐层展开，不要求一开始就填满：

```
阶段级（初始，main 创建）：
  #4 coder: 架构设计 → 🔄
  #6 coder: 编码实现 → ⬜

子任务级（agent 自行细化）：
  #4 coder: 架构设计 → 🔄
    #4.1 数据模型设计 → ⬜
    #4.2 API 接口设计 → ⬜
    #4.3 可测试项清单 → ⬜
  #6 coder: 编码实现 → ⬜
```

---

## 谁可以建任务

| 场景 | 建任务者 | 承接者 | 示例 |
|------|---------|--------|------|
| main 派活 | main | 任意 agent | `#4 coder: 架构设计` |
| agent 细化 | agent 自己 | 自己 | `#4.1 coder: 数据模型设计` |
| agent 请求 | agent | 其他 agent | `#6.1 tester: 性能压测` |

## 闭环子任务模板

> agent 进入以下阶段时，将对应模板展开为子任务，追踪闭环全过程。

### 🔧 coder 自测闭环（阶段6）

```
#6 coder: 编码实现 → 🔄
  #6.1 编码 → ⬜
  #6.2 自测 → ⬜
    #6.2.1 发现: [问题描述] → ⬜
    #6.2.2 修复: [方案] → ⬜
    #6.2.3 自测回归 → ⬜
  #6.3 交付 tester → ⬜
```

**规则：** coder 自测发现问题自行修复，不经过 tester。所有自测问题修复完成后才能交付。

### 🧪 tester 缺陷闭环（阶段7）

```
#7 tester: 测试 → 🔄
  #7.1 冒烟测试 → ⬜
  #7.2 黑盒用例设计 → ⬜
  #7.3 白盒用例设计 → ⬜
  #7.4 测试执行 → ⬜
  #7.5 缺陷管理 (迭代第1轮) → ⬜
    #7.5.1 [BUG-001] 登记 → ⬜
    #7.5.2 [BUG-001] coder 修复 → ⬜
    #7.5.3 [BUG-001] tester 验证 → ⬜
    #7.5.4 [BUG-001] 关闭 / 重开 → ⬜
  #7.6 缺陷管理 (迭代第2轮, 如有) → ⬜
  #7.7 回归测试 → ⬜
  #7.8 测试报告 → ⬜
```

**规则：**
- 每轮迭代追加子任务，上限 3 轮
- 首次 bug 和最终通过时告知 main
- 超限 main 介入裁决

### 🔒 auditor 问题闭环（阶段2/8）

```
#2 / #8 auditor: 审计 → 🔄
  #2.1 / #8.1 证据收集 → ⬜
  #2.2 / #8.2 审计执行 → ⬜
  #2.3 问题管理 → ⬜
    #2.3.1 [AUDIT-001] 登记 → ⬜
    #2.3.2 [AUDIT-001] 报告 main → ⬜
    #2.3.3 [AUDIT-001] main 决策 (修复/忽略/技术债务) → ⬜
    #2.3.4 [AUDIT-001] coder 修复 → ⬜
    #2.3.5 [AUDIT-001] tester 验证 → ⬜
    #2.3.6 [AUDIT-001] auditor 复核 → ⬜
    #2.3.7 [AUDIT-001] main 裁决闭环 → ⬜
  #2.4 / #8.4 审计报告 → ⬜
```

**规则：**
- 每个问题走完 登记→跟踪→验证→闭环 七步
- 闭环裁决权在 main（auditor 只发现问题，不裁决）
- checklist 未 100% 通过不得放行

## 断点再续

项目中断后，todo.md 是唯一恢复依据：

```
最后一个完成的 todo: #5 analyze ✅
下一个待开始的 todo: #6 编码实现 ⬜ (承接: coder)
→ main 直接会话 coder: "继续项目X，阶段6编码，分支 feature/xxx"
```
